Enhance DQN (PyTorch Lightning) 特色與架構說明

封裝與模組化 (Encapsulation & Modularity):
訓練邏輯與 Web 介面完全解耦。train_offline.py 是一支可獨立執行的 Python 腳本，在終端機直接執行 python train_offline.py 即可完整訓練模型，無需啟動 Flask 伺服器。訓練完畢後模型權重儲存為 trained_lightning.pth；Flask 後端 (app.py) 僅在使用者點擊驗證時載入此檔案進行推論，實現了「訓練」與「部署驗證」的完全分離。PyTorch Lightning 將訓練迴圈、優化器更新、LR Scheduler 與硬體加速邏輯統一封裝於 LitDQN_PER 模組，大幅減少樣板程式碼 (Boilerplate)。

隨機模式 (Random Mode):
在每一局訓練開始時，環境中的起點 (P)、終點 (+)、陷阱 (-) 以及牆壁 (W) 皆會隨機生成。這迫使模型不能死背路線，而是必須學會「根據當前看到的畫面找出最佳策略」，大幅提升模型的泛化能力 (Generalization)。

訓練穩定技巧 (Training Tips):
1. Gradient Clipping (梯度裁剪): 設定 gradient_clip_val=1.0，防止在面對隨機環境的意外狀況時，梯度過大導致神經網路權重崩潰。
2. Learning Rate Scheduling (學習率排程): 使用 ExponentialLR 讓學習率隨時間遞減，前期大步探索，後期穩定收斂。

Double DQN

方法特色與架構說明

Formula: y = r + γ Qtarget(s', argmaxa' Qonline(s', a'; θ); θ')

設計方式:
將動作的「選擇」與「評估」解耦。使用 Online Network 選擇下一個狀態的最佳動作，再用 Target Network 評估該動作的 Q 值，緩解 Vanilla DQN 的 Q-value Overestimation。

網路架構:
Linear(64 → 150) → ReLU → Linear(150 → 100) → ReLU → Linear(100 → 4)。 Online 與 Target 同形狀；Target 每 500 環境步同步一次。

離線訓練超參數:

Dueling DQN

方法特色與架構說明

Formula: Q(s,a) = V(s) + (A(s,a) - 1/|A|∑A(s,a'))

設計方式:
特徵擷取後分成 Value Stream V(s) 與 Advantage Stream A(s,a)，再合併為 Q。 讓模型在「動作不影響結果」的狀態下能單獨學狀態價值。

網路架構:
Shared feature: Linear(64 → 150) → ReLU
Advantage head: Linear(150 → 100) → ReLU → Linear(100 → 4)
Value head:    Linear(150 → 100) → ReLU → Linear(100 → 1)
Target rule：vanilla（max over Target Network）— 隔離「架構」對結果的貢獻。

離線訓練超參數:（與 Double DQN 共用，僅 target rule 與網路架構不同）

項目設定
Epochs1500
Max moves / episode50
Batch size200
Replay bufferdeque(10,000)，uniform sampling
Discount γ0.9
OptimizerAdam, lr=1e-3
LR schedulerCosineAnnealingLR (1e-3 → 1e-5)
ε-greedy1.0 → 0.05，前 70% epochs 線性衰減
Target network sync每 500 環境步

Rainbow DQN — 六合一強化的離線預訓練模型

Pre-trained Loaded

本模式 不進行線上訓練。Rainbow DQN 已由 train_offline_rainbow.py 以 4-stage curriculum（4×4 → 4-7 矩形混合）離線訓練完成， 權重存於 trained_rainbow.pth， Flask 後端僅在點擊驗證時推論。

✓ Double DQN ✓ Dueling Network ✓ Prioritized Replay (PER) ✓ Multi-step Returns (n=3) ⚡ NoisyNet (replaces ε-greedy) ✓ Distributional RL (C51)

網路架構： CNN trunk 共享特徵後分流到 Value / Advantage 兩個 NoisyLinear head； 每個動作輸出一個 51-atom 機率分佈（Vmin=−10, Vmax=+10）， 最終 Q(s,a) = Σi zi·pi。

泛化亮點： 單一網路支援所有 4≤X≤7、4≤Y≤7 的矩形網格（16 種尺寸）， 輸入 zero-pad 到 7×7，CNN 的 translation invariance 讓模型自動處理 (X,Y) ↔ (Y,X) 對稱。

互動驗證： 可在下方選擇尺寸並拖曳 Player / Goal / Pit / Wall 任意位置，按 🧠 Run Verification 後模型會以確定性 greedy policy 走出最佳路徑。
