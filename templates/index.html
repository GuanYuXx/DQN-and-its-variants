HW3-1: Naive DQN（static mode）— 方法與超參說明

問題動機 / Motivation： DRL in Action 程式 3.5 的 vanilla DQN 用同一個 online net 同時計算 Q(s,a) 與 bootstrap max Q(s',a')，目標值會隨著梯度更新而漂移，造成 moving target 振盪。 HW3-1 在原 Experience Replay 之上加入 Target Network（程式 3.7/3.8）：用 copy.deepcopy(model) 凍結一份權重專門算 bootstrap value，每 sync_freq 步硬同步一次，大幅穩定 Q-value 估計。

Target rule（含 Target Network）
$$y \;=\; r + \gamma \cdot \max_{a'} Q_{\theta^{-}}(s', a')\qquad\text{Loss}\;=\;\bigl(\,Q_{\theta}(s,a) - y\,\bigr)^2$$

HW3-3: Enhance DQN (PyTorch Lightning) 特色與架構說明

問題動機 / Motivation： Random mode 每個 episode 把 P/G/Pit/W 全部洗牌，模型 不能死背，必須從「當前畫面」推路徑。 為了應付這種高方差環境，本實作把多種穩定技巧整合到 PyTorch Lightning 模組 LitDQN_PER 內：PER（SumTree-based proportional priority，把高 TD-error sample 多看幾次）、Double target rule、CosineAnnealingLR（前期大步、後期穩定）、Gradient Clipping（防爆炸）、Target Network step-based sync。

PER：以 |TD-error| 為優先度抽樣（α=0.6），用 IS-weight 還原無偏估計（β: 0.4→1.0）
$$P(i) \;=\; \frac{p_i^{\alpha}}{\sum_k p_k^{\alpha}}\,,\quad w_i \;=\; \Bigl(\frac{1}{N\cdot P(i)}\Bigr)^{\!\beta}\,,\quad \mathcal{L} \;=\; \tfrac{1}{B}\sum_i w_i \cdot \bigl(Q_{\theta}(s_i,a_i) - y_i\bigr)^2$$

PyTorch Lightning 封裝：訓練 loop / optimizer / scheduler / 硬體加速統一由 LitDQN_PER 管理，部署端 app.py 只負責推論。

封裝與模組化 (Encapsulation & Modularity):
訓練邏輯與 Web 介面完全解耦。train_offline.py 是一支可獨立執行的 Python 腳本，在終端機直接執行 python train_offline.py 即可完整訓練模型，無需啟動 Flask 伺服器。訓練完畢後模型權重儲存為 trained_lightning.pth；Flask 後端 (app.py) 僅在使用者點擊驗證時載入此檔案進行推論，實現了「訓練」與「部署驗證」的完全分離。PyTorch Lightning 將訓練迴圈、優化器更新、LR Scheduler 與硬體加速邏輯統一封裝於 LitDQN_PER 模組，大幅減少樣板程式碼 (Boilerplate)。

隨機模式 (Random Mode):
在每一局訓練開始時，環境中的起點 (P)、終點 (+)、陷阱 (-) 以及牆壁 (W) 皆會隨機生成。這迫使模型不能死背路線，而是必須學會「根據當前看到的畫面找出最佳策略」，大幅提升模型的泛化能力 (Generalization)。

訓練穩定技巧 (Training Tips):
1. Gradient Clipping (梯度裁剪): 設定 gradient_clip_val=1.0，防止在面對隨機環境的意外狀況時，梯度過大導致神經網路權重崩潰。
2. Learning Rate Scheduling (學習率排程): 使用 CosineAnnealingLR 讓學習率隨時間遞減，前期大步探索，後期穩定收斂。

★ 相對 starter spec 的加分項
HW3-3 規格的必要項只有「將 PyTorch DQN 轉成 Lightning（或 Keras）」，並 bonus 提到 gradient clipping / LR scheduling。本實作（train_offline.py + LitDQN_PER）額外加入：
  • Prioritized Experience Replay (PER) — SumTree-based O(log N) 抽樣，α=0.6，IS weight β 0.4→1.0 線性退火（不在 HW3-3 必要清單）
  • Double DQN target rule — y = r + γ · Qtarget(s', argmax Qonline(s'))，減少高估偏差
  • Target Network — 每 500 環境步同步（step-based，比 epoch-based 更穩定）
  • CosineAnnealingLR — 1e-3 → 1e-5 over 5000 epochs（取代 ExponentialLR）
  • Gradient clipping — clip_grad_norm_ ≤ 1.0
  • ε 下限 0.05（starter 為 0.1）+ 指數衰減 × 0.99997 / step
  • 放大 replay buffer 至 20,000（starter 為 1,000）、batch 256（starter 為 200）
  • Random mode 環境擴充：starter 永遠固定 1 Pit + 1 Wall；本實作 obstacle 數量隨機介於 1 ≤ num_pits + num_walls ≤ min(W,H)−1， 讓模型暴露於更廣的障礙密度分佈、提升泛化能力

HW3-2 · 離線預訓練模型 — 互動驗證

已載入預訓練模型

問題動機 / Motivation： Vanilla DQN 用同一 net 同時「選 argmax 動作」與「估那個動作的 Q 值」，使得 max 算子帶來系統性 Q-value overestimation。Double DQN 把「選」與「估」拆給 online / target 兩個 net；Dueling 則改網路架構，把 trunk 後分成 V(s)（與動作無關的狀態價值）和 A(s, a)（動作優勢），再 mean-subtract 合併為 Q——讓網路在「動作不影響結果」的狀態下能單獨學狀態價值。

Double DQN target rule（動作選擇 vs 估值解耦）
$$y \;=\; r + \gamma \cdot Q_{\theta^{-}}\!\Bigl(s',\;\arg\max_{a'} Q_{\theta}(s', a')\Bigr)$$
Dueling decomposition（mean-subtraction 確保 V/A 可辨識）
$$Q(s,a) \;=\; V(s) \;+\; \Bigl(A(s,a) \;-\; \tfrac{1}{|\mathcal{A}|}\!\sum_{a'} A(s, a')\Bigr)$$

Double DQN

方法特色與架構說明

Formula: y = r + γ Qtarget(s', argmaxa' Qonline(s', a'; θ); θ')

設計方式:
將動作的「選擇」與「評估」解耦。使用 Online Network 選擇下一個狀態的最佳動作，再用 Target Network 評估該動作的 Q 值，緩解 Vanilla DQN 的 Q-value Overestimation。

網路架構:
Linear(64 → 150) → ReLU → Linear(150 → 100) → ReLU → Linear(100 → 4)。 Online 與 Target 同形狀；Target 每 500 環境步同步一次。

Dueling DQN

方法特色與架構說明

Formula: Q(s,a) = V(s) + (A(s,a) - 1/|A|∑A(s,a'))

設計方式:
特徵擷取後分成 Value Stream V(s) 與 Advantage Stream A(s,a)，再合併為 Q。 讓模型在「動作不影響結果」的狀態下能單獨學狀態價值。

網路架構:
Shared feature: Linear(64 → 150) → ReLU
Advantage head: Linear(150 → 100) → ReLU → Linear(100 → 4)
Value head:    Linear(150 → 100) → ReLU → Linear(100 → 1)
Target rule：vanilla（max over Target Network）— 隔離「架構」對結果的貢獻。

HW3-4 · Rainbow DQN — 六合一強化的離線預訓練模型

Pre-trained Loaded

問題動機 / Motivation： Rainbow (Hessel et al. 2017) 整合 6 種獨立改進；本實作再加上 CNN trunk + 可變網格 4×4 ~ 7×7 + 4-stage curriculum。 最關鍵：用 C51 distributional 預測整個回報分佈而非單一期望值，再用 NoisyNet 取代 ε-greedy（每次 forward 自動 reset_noise 探索）。

Distributional Q（C51, 51 atoms）— 用機率分佈代替點估計
$$Q(s,a) \;=\; \sum_{i=0}^{N-1} z_i \cdot p_i(s,a)\,,\quad z_i = V_{\min} + i\cdot\Delta z\,,\quad (V_{\min}, V_{\max}, N) = (-10,\,+10,\,51)$$
NoisyLinear（factorised Gaussian）— 把 ε-greedy 內化進權重
$$y \;=\; (\mu^{w} + \sigma^{w}\!\odot\!\varepsilon^{w})\,x \;+\; (\mu^{b} + \sigma^{b}\!\odot\!\varepsilon^{b})\,,\quad \varepsilon^{w}_{i,j} = f(\varepsilon_i)\,f(\varepsilon_j)$$

CNN trunk 共享空間特徵，Dueling V/A 在 per-atom 層級組合（先合分佈再 dueling-mean 校正），最後 softmax 得 51-atom 機率，乘以 atom 值求出 Q。

本模式 不進行線上訓練。Rainbow DQN 已由 train_offline_rainbow.py 以 4-stage curriculum（4×4 → 4-7 矩形混合）離線訓練完成， 權重存於 trained_rainbow.pth， Flask 後端僅在點擊驗證時推論。

✓ Double DQN ✓ Dueling Network ✓ Prioritized Replay (PER) ✓ Multi-step Returns (n=3) ⚡ NoisyNet (replaces ε-greedy) ✓ Distributional RL (C51)

網路架構： CNN trunk 共享特徵後分流到 Value / Advantage 兩個 NoisyLinear head； 每個動作輸出一個 51-atom 機率分佈（Vmin=−10, Vmax=+10）， 最終 Q(s,a) = Σi zi·pi。

泛化亮點： 單一網路支援所有 4≤X≤7、4≤Y≤7 的矩形網格（16 種尺寸）， 輸入 zero-pad 到 7×7，CNN 的 translation invariance 讓模型自動處理 (X,Y) ↔ (Y,X) 對稱。

互動驗證： 可在下方選擇尺寸並拖曳 Player / Goal / Pit / Wall 任意位置，按 🧠 Run Verification 後模型會以確定性 greedy policy 走出最佳路徑。

★ 相對 starter spec 的加分項（整題為加分題）
HW3-4 加分題的需求僅為「使用 Rainbow DQN 解 Random Mode GridWorld」。本實作除了 Rainbow 六大 component 之外，還做了以下額外擴充：
  • CNN trunk（Conv 4→32→64 + FC 3136→512）取代 starter 的純 MLP — 利用 spatial locality 學習障礙物 / 目標的相對位置
  • 支援可變尺寸網格：單一網路覆蓋 4×4 ∼ 7×7 共 16 種 (X,Y) 組合，輸入 zero-pad 到 7×7，CNN translation invariance 同時學到 (X,Y) ↔ (Y,X) 對稱性
  • 4-stage curriculum learning — 按 max(X,Y) 由小到大漸進，並保留前一階段 replay 做 rehearsal，緩解 catastrophic forgetting
  • Random mode 障礙物全隨機：每 episode Goal / Pit / Wall / Player 四件全隨機 + 不重疊，並用 BFS 預檢確保可達（starter 為固定 Wall + Goal + Pit）
  • NoisyNet head-only：僅最後 4 個 Linear 換成 NoisyLinear，前段共享 trunk 維持普通 Linear — 降低參數量、訓練更穩
  • 互動驗證 UI：可拖曳 Player / Goal / Pit / Wall 任意佈局，推論時 disable_noise() 確保 deterministic greedy 路徑
