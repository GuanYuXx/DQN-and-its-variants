Enhance DQN (PyTorch Lightning) 特色與架構說明

封裝與模組化 (Encapsulation & Modularity):
訓練邏輯與 Web 介面完全解耦。train_offline.py 是一支可獨立執行的 Python 腳本，在終端機直接執行 python train_offline.py 即可完整訓練模型，無需啟動 Flask 伺服器。訓練完畢後模型權重儲存為 trained_lightning.pth；Flask 後端 (app.py) 僅在使用者點擊驗證時載入此檔案進行推論，實現了「訓練」與「部署驗證」的完全分離。PyTorch Lightning 將訓練迴圈、優化器更新、LR Scheduler 與硬體加速邏輯統一封裝於 LitDQN_PER 模組，大幅減少樣板程式碼 (Boilerplate)。

隨機模式 (Random Mode):
在每一局訓練開始時，環境中的起點 (P)、終點 (+)、陷阱 (-) 以及牆壁 (W) 皆會隨機生成。這迫使模型不能死背路線，而是必須學會「根據當前看到的畫面找出最佳策略」，大幅提升模型的泛化能力 (Generalization)。

訓練穩定技巧 (Training Tips):
1. Gradient Clipping (梯度裁剪): 設定 gradient_clip_val=1.0，防止在面對隨機環境的意外狀況時，梯度過大導致神經網路權重崩潰。
2. Learning Rate Scheduling (學習率排程): 使用 ExponentialLR 讓學習率隨時間遞減，前期大步探索，後期穩定收斂。

Double DQN

方法特色與架構說明

Formula: y = r + γ Qtarget(s', argmaxa' Qonline(s', a'; θ); θ')

設計方式 (Design Method):
Double DQN 將動作的「選擇」與「評估」解耦。使用 Online Network 來選擇下一個狀態的最佳動作，然後使用 Target Network 來評估這個動作的 Q 值。這能有效解決基礎 DQN 中經常發生的 Q 值過度高估 (Overestimation) 問題。

流程架構:
狀態 s' → Online Network 選擇最佳動作 a* → Target Network 評估該動作價值 → 計算目標 Q 值。

Dueling DQN

方法特色與架構說明

Formula: Q(s,a) = V(s) + (A(s,a) - 1/|A|∑A(s,a'))

設計方式 (Design Method):
Dueling DQN 改變了神經網路的內部架構，在特徵擷取後，將資料流分為兩支：一支計算狀態本身的價值 V(s) (State Value)，另一支計算各動作的相對優勢 A(s,a) (Advantage)，最後再合併。這讓模型在某些「動作不影響最終結果」的狀態下，能更專注且準確地學習狀態價值。

流程架構:
狀態 s → 共用特徵擷取 (Shared Feature Extractor) → 分流至 Value Stream 與 Advantage Stream → 合併輸出 Q(s,a)。
